🔍 Geocoding

Geocoding

POST
/api/geocode
Volltext-Adresssuche via Nominatim — mit POI-Fallback bei Kategoriesuche
Request-Body
query*stringSuchbegriff — Adresse, Ort, POI-Name oder "apotheke köln"
limit optint 1–25Maximale Trefferzahl (default: 10)
Response
resultsarrayTreffer mit lat, lon, display_name, address
sourcestringnominatim oder poi-fallback
curl -X POST BASE/api/geocode \ -H 'Content-Type: application/json' \ -d '{"query": "Blütenstr. 62 47166 Duisburg", "limit": 5}' // Response { "results": [{ "lat": "51.5136", "lon": "6.7921", "display_name": "Blütenstraße 62, Duisburg, NRW, ...", "address": { "road": "Blütenstraße", "postcode": "47166", "city": "Duisburg" } }], "source": "nominatim" }
POST
/api/geocode/suggest
Live-Autocomplete für Suchfelder — niedriges Limit, kurzer Timeout, ab 2 Zeichen
Request-Body
query*string ≥ 2Teiladresse — bei < 2 Zeichen gibt der Endpoint direkt [] zurück
limit optint 1–8Maximale Trefferzahl (default: 5)
Frontend-Pattern: Debounce 250–300 ms empfohlen. Jeder Keystroke ohne Debounce trifft Nominatim — das kostet Performance.
// Suchfeld mit Autocomplete (JS) let _timer; searchInput.addEventListener('input', () => { clearTimeout(_timer); _timer = setTimeout(async () => { const res = await fetch('BASE/api/geocode/suggest', { method: 'POST', headers: {'Content-Type': 'application/json'}, body: JSON.stringify({query: searchInput.value, limit: 5}) }); const {results} = await res.json(); renderDropdown(results); // [{lat, lon, display_name, ...}] }, 280); });
POST
/api/reverse
Reverse-Geocoding — Koordinaten → Adresse via Nominatim
Request-Body
lat*floatBreitengrad
lon*floatLängengrad
zoom optint 0–18Detailstufe — 18 = Hausnummer (default), 16 = Straße, 12 = Ort
curl -X POST BASE/api/reverse \ -H 'Content-Type: application/json' \ -d '{"lat": 51.5136, "lon": 6.7921, "zoom": 18}'
POST
/api/distance
Haversine-Distanz + Kompass-Peilung zwischen zwei Punkten — kein externer Service nötig
Request-Body
from*[lat, lon]Startpunkt als Array
to*[lat, lon]Zielpunkt als Array
Response
distance_mintLuftlinien-Distanz in Metern
distance_kmfloatLuftlinien-Distanz in Kilometern
bearing_degfloatPeilung in Grad (0° = Nord, 90° = Ost, 180° = Süd, 270° = West)
bearing_compassstringN / NE / E / SE / S / SW / W / NW
curl -X POST BASE/api/distance \ -H 'Content-Type: application/json' \ -d '{"from": [51.5136, 7.4653], "to": [50.9413, 6.9583]}' // Response { "distance_m": 67482, "distance_km": 67.482, "bearing_deg": 218.4, "bearing_compass": "SW", "from": {"lat": 51.5136, "lon": 7.4653}, "to": {"lat": 50.9413, "lon": 6.9583} }
🚗 Routing

Routenplanung

POST
/api/route
Multi-Stop-Routing via GraphHopper — Geometrie, Distanz, Zeit, Abbiegehinweise
Request-Body
profile*stringcar | foot | bike
points*[[lat,lon]]Mindestens zwei Wegpunkte — beliebig viele Zwischenstopps möglich
Response
distance_mfloatStreckenlänge in Metern
time_msintFahrzeit in Millisekunden
geometryGeoJSONLineString — direkt als MapLibre/Leaflet Source verwendbar
instructionsarrayAbbiegehinweise mit text, distance, sign
curl -X POST BASE/api/route \ -H 'Content-Type: application/json' \ -d '{"profile": "car", "points": [[51.5136, 7.4653], [50.9413, 6.9583]]}'
POST
/api/isochrone
Reichweiten-Isochrone via ORS — "Was erreiche ich in X Minuten?" als GeoJSON-Polygon
Request-Body
lat*floatStartpunkt Breitengrad
lon*floatStartpunkt Längengrad
profile optstringcar | foot | bike (default: car)
range opt[int]Reichweiten-Stufen — Sekunden bei time, Meter bei distance (default: [300] = 5 Min.)
range_type optstringtime (default) | distance — max. 6 Stufen pro Aufruf
Response: GeoJSON FeatureCollection direkt von ORS — jedes Feature enthält ein Polygon mit der Reichweite. Mehrere Stufen in einem Aufruf möglich: "range": [300, 600, 900] = 5/10/15 Min.
# 5 und 10 Minuten mit dem Auto curl -X POST BASE/api/isochrone \ -H 'Content-Type: application/json' \ -d '{"lat": 51.4818, "lon": 7.2162, "profile": "car", "range": [300, 600]}' # 2 km Fußweg-Radius curl -X POST BASE/api/isochrone \ -H 'Content-Type: application/json' \ -d '{"lat": 51.4818, "lon": 7.2162, "profile": "foot", "range": [2000], "range_type": "distance"}'
POST
/api/matrix
Distanz-Matrix via GraphHopper — alle Distanzen/Zeiten zwischen n Ursprüngen und m Zielen
Request-Body
profile*stringcar | foot | bike
origins*[[lat,lon]]Startpunkte — max. 10
destinations opt[[lat,lon]]Zielpunkte (default: = origins → quadratische Matrix)
Limit: 10 × 10 = 100 Kombinationen — bei Überschreitung HTTP 400.
Response
distances[[float]]Matrix Distanzen in Metern — distances[i][j] = von Origin i nach Destination j
times[[int]]Matrix Zeiten in Millisekunden
curl -X POST BASE/api/matrix \ -H 'Content-Type: application/json' \ -d '{"profile": "car", "origins": [[51.5136, 7.4653], [50.9413, 6.9583]], "destinations": [[51.2277, 6.7735]]}' // Response — distances[i][j] in Metern { "distances": [[52341], [41222]], "times": [[2340000], [1980000]], "origins_count": 2, "destinations_count": 1 }
📍 POI

Points of Interest

POST
/api/poi/radius
Umkreissuche — POIs innerhalb eines echten Haversine-Radius, sortiert nach Entfernung
Request-Body
category*stringPOI-Kategorie-Key (siehe Kategorien)
lat*floatMittelpunkt Breitengrad
lon*floatMittelpunkt Längengrad
radius_km optfloat 0.1–25Suchradius in Kilometern (default: 1.0)
limit optint 1–50Max. Trefferzahl (default: 20)
Response — jedes Ergebnis enthält zusätzlich
distance_mintHaversine-Distanz zum Mittelpunkt in Metern
name / tagsstring/objName und OSM-Tags (Adresse, Öffnungszeiten, …)
curl -X POST BASE/api/poi/radius \ -H 'Content-Type: application/json' \ -d '{"category": "pharmacy", "lat": 51.4818, "lon": 7.2162, "radius_km": 2.0}' // Response { "category": "pharmacy", "label": "Apotheke", "center": {"lat": 51.4818, "lon": 7.2162}, "radius_km": 2.0, "count": 7, "results": [ {"name": "Apotheke am Markt", "distance_m": 423, "lat": 51.4801, "lon": 7.2194, "tags": {"addr:street": "Marktplatz", ...}}, ... ] }
POST
/api/poi
POI-Suche in einer Bounding Box (Rechteck) via Overpass
Request-Body
category*stringPOI-Kategorie-Key
bbox*[s,w,n,e]Bounding Box [south, west, north, east] — max. 4 Grad²
curl -X POST BASE/api/poi \ -H 'Content-Type: application/json' \ -d '{"category": "restaurant", "bbox": [51.45, 7.20, 51.52, 7.26]}'
GET
/api/poi/categories
Alle verfügbaren POI-Kategorien mit Label
{% set cats = [ ("pharmacy","Apotheke"),("restaurant","Restaurant"),("cafe","Café"),("bar","Bar"),("pub","Kneipe"), ("fuel","Tankstelle"),("bank","Bank"),("atm","Geldautomat"),("hospital","Krankenhaus"), ("doctors","Arzt"),("school","Schule"),("supermarket","Supermarkt"),("bakery","Bäckerei"),("convenience","Kiosk") ] %} {% for key, label in cats %} {{ key }} {{ label }} {% endfor %}
curl BASE/api/poi/categories [{"key":"pharmacy","label":"Apotheke"}, {"key":"restaurant","label":"Restaurant"}, ...]
